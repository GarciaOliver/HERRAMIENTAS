CARLOS:¿Qué comando se usa para clonar un repositorio remoto?
git clone con la url del repositorio
CARLOS:¿Cómo se crea una nueva rama en Git?
Se debe utilizar el comando git checkout -b "Nombre de la rama"
CARLOS:¿Qué hace el comando git status?
El comando git status muestra el estado actual del repositorio en Git, incluyendo información sobre:
Cambios en el área de trabajo: Archivos modificados, nuevos o eliminados que aún no han sido añadidos al área de preparación (staging area).
Cambios en el área de preparación: Archivos que ya han sido añadidos al área de preparación y están listos para ser confirmados (committed).
Archivos sin seguimiento (untracked): Archivos nuevos que Git detecta en el directorio de trabajo pero que aún no están siendo rastreados.
Conflictos: En caso de un conflicto de merge, muestra los archivos con conflictos pendientes de resolución.
CARLOS:¿Cómo fusionas una rama con la principal?
Para fusionar una rama con la principal en Git, se utiliza el comando git merge.
CARLOS:¿Qué diferencia hay entre git pull y git fetch?
git fetch
Descarga las actualizaciones del repositorio remoto (commits, ramas, etiquetas, etc.) y las guarda en tu repositorio local.
No realiza ninguna fusión ni modifica tu rama actual automáticamente.
Es útil para revisar los cambios remotos antes de integrarlos.

git pull
Es una combinación de dos comandos: git fetch (descargar los cambios) seguido de git merge (fusionar los cambios remotos con tu rama actual).
Actualiza tu rama de trabajo directamente al traer los cambios del remoto.
